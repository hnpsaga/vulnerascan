# Dashboard

The VulneraScan dashboard provides a browser-based view of your projects, scan history, vulnerabilities, and generated reports.

Every successful scan is automatically added to the dashboard. No additional configuration or import step is required.

Launch the dashboard with:

```bash
vulnerascan dashboard
```

By default, the dashboard is available at:

```text
http://localhost:4000
```

---

# Getting Started

1. Scan one or more projects.

```bash
vulnerascan scan
```

2. Start the dashboard.

```bash
vulnerascan dashboard
```

3. Open the displayed URL in your browser if it does not open automatically.

The dashboard immediately displays every project that has been scanned using VulneraScan.

---

# Dashboard Overview

The home page provides a high-level summary of your local workspace.

It includes:

* Total projects
* Total scans
* Total dependencies
* Total vulnerabilities
* Latest scan timestamp
* Vulnerabilities by severity
* Projects grouped by ecosystem
* Recent scan history

Use the ecosystem and severity filters to focus on a subset of projects.

---

# Projects

The Projects view lists every project that has been scanned.

For each project you can view:

* Project name
* Supported ecosystem
* Current status
* Vulnerability summary
* Dependency count
* Last scan time

Projects can be searched by name and filtered by ecosystem or severity.

Selecting a project opens the detailed project view.

---

# Vulnerabilities

The Vulnerabilities view provides a workspace-wide summary of detected vulnerabilities.

It allows you to:

* Search findings
* Filter by ecosystem
* Filter by severity
* Navigate directly to the affected project's detailed results

---

# Scan History

Every scan is preserved as a separate historical record.

The History view displays:

* Scan identifier
* Project
* Scan time
* Ecosystem
* Scan status
* Vulnerability count

Running another scan does not replace previous results. Each scan becomes part of the project's history, allowing you to compare results over time.

---

# Project Details

Selecting a project displays detailed scan information, including:

* Project information
* Scan summary
* Dependency statistics
* Vulnerability findings
* Severity information
* Advisory details
* Historical scan timeline

Each historical scan can be opened independently.

---

# Download Reports

From the project details page you can download the reports generated for a scan.

Available formats include:

* JSON
* Markdown
* CSV
* SARIF
* CycloneDX SBOM
* SPDX SBOM
* AI Context (JSON)
* AI Context (Markdown)

These are the same reports generated by the CLI.

# Dashboard Commands

## Start the Dashboard

```bash
vulnerascan dashboard
```

### Options

| Option              | Description                                                                  |
| ------------------- | ---------------------------------------------------------------------------- |
| `-p, --port <port>` | Dashboard server port. Default: `4000`.                                      |
| `-h, --host <host>` | Host interface. Defaults to `localhost` or `VULNERASCAN_HOST` if configured. |

---

## Workspace Summary

Display a summary directly in the terminal:

```bash
vulnerascan dashboard summary
```

Available filters:

* `--ecosystem`
* `--severity`
* `--start-date`
* `--end-date`

---

## Projects Summary

List registered projects from the terminal:

```bash
vulnerascan dashboard projects
```

Available filters:

* `--ecosystem`
* `--severity`

---

# Configuration

The dashboard can be configured using command-line options or environment variables.

| Setting | Default     |
| ------- | ----------- |
| Host    | `localhost` |
| Port    | `4000`      |

Environment variables:

* `VULNERASCAN_HOST`
* `VULNERASCAN_HOME`

---

# Docker

The dashboard is fully supported when running VulneraScan through Docker.

Expose the dashboard port when starting the container:

```bash
docker run --rm \
  -p 4000:4000 \
  -v ~/.vulnerascan:/root/.vulnerascan \
  ghcr.io/hnpsaga/vulnerascan:latest \
  dashboard
```

When running inside Docker, VulneraScan does not attempt to open a browser automatically. Open the displayed URL manually from your host machine.

# Typical Workflow

A common workflow is:

1. Scan one or more projects.
2. Open the dashboard.
3. Review project summaries.
4. Investigate vulnerabilities.
5. Download reports if required.
6. Re-scan projects after applying fixes.

Because every scan is retained, the dashboard provides a historical view of your project's security posture over time.
